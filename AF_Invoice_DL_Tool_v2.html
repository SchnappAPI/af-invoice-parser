AppFolio Invoice Attachment Processor

Bulk renames and downloads payable invoice attachments
⚠
Firefox only. This tool is not configured to work in Google Chrome or any other browser. Open this page and run the processor in Firefox.
Required CSV columns: Txn ID and Reference. Missing either column will cause an error.
Note: This tool is designed specifically for AppFolio Accounting → Payable Invoices. It has not been tested against other areas of AppFolio and may not work correctly outside of that context.
Step 1 — Install
1
Drag the bookmarklet to your Firefox bookmark bar
This is a one-time setup. Once it is saved, click it from any AppFolio page to launch the panel.
AF Invoice DL drag this to your bookmark bar
Step 2 — Run
1
Allow popups for AppFolio
The first time you run, Firefox will block popups. Click the notification in the address bar and choose Always allow popups for firstlake.appfolio.com. You only need to do this once.
2
Export your CSV from AppFolio
Open your bill report. Click Columns and make sure Txn ID and Reference are visible. Then export using Actions → Export as CSV.
Note: Txn ID is hidden by default on some reports. If it is not visible, use the Columns button to add it.
3
Launch the panel and load your CSV
Navigate to any AppFolio page in Firefox. Click AF Invoice DL in your bookmark bar. In the panel, click Choose File and select your CSV.
Returning users: The panel restores your saved progress automatically. You do not need to reload the CSV unless you are starting a new job.
4
Set batch size and click Run Batch
A batch size of 25 to 50 is recommended. The processor opens each invoice, renames the attachment to the Reference value, downloads it, and moves on. Progress is saved after every invoice so you can stop and resume at any time.
Keep the AppFolio tab active and visible while the batch is running.
Log Statuses
?
OK SKIP ERROR
  • OK — attachment renamed and downloaded successfully.
  • SKIP — invoice had no attachment. Not counted as a failure.
  • ERROR — something failed. The log shows the reason. Use Reset Progress to reprocess from scratch.
